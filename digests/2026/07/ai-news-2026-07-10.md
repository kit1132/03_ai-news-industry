## 最重要：OpenAI「ChatGPT Work」投入 — Codex内蔵の常駐エージェントが業務を完遂、Claude Cowork に直接対抗

- **アプリ横断で「完成品」を数時間かけて生成（Bloomberg / MacRumors / 9to5Mac、7/9）**: ChatGPT Work は Codex を内蔵したエージェントで、接続済みアプリから文脈を収集 → ゴールをステップ分解 → スプレッドシート/スライド/ドキュメント/Webアプリを完成させる。大きなプロジェクトを小タスクに分解して数時間連続稼働、スケジュール実行で自律動作。Web/モバイル/デスクトップ横断で開始・管理可能（スマホからも起動）。実質的に Claude Cowork の対抗製品。
  - https://www.bloomberg.com/news/articles/2026-07-09/openai-unveils-chatgpt-work-agent-to-field-tasks-for-hours
  - https://www.macrumors.com/2026/07/09/openai-chatgpt-work/
- **エンタープライズ統制を標準装備**: 管理者は接続ツール・社内データ・実行アクションを中央管理。**Compliance API** で ChatGPT Work の会話・アクションを可視化、機微なアクションは実行前に自動レビューでチェック。Pro/Enterprise/Edu で本日提供、数日中に Plus/Business へ拡大。あわせて Codex 単体アプリを ChatGPT デスクトップアプリに統合（7/9〜）。
  - https://9to5mac.com/2026/07/09/openai-announcing-the-next-chapter-for-chatgpt-today-watch-here/
- **コンサル観点**: 「チャットで答える AI」から「業務成果物を納品する常駐エージェント」への移行を、OpenAI・Anthropic の両巨頭が同型プロダクト（ChatGPT Work / Claude Cowork）で正面から競う構図が確定。**導入検討では compliance/監査 API とアクション事前レビューの有無が実質的な選定基準になる**。M365 Copilot・Claude Cowork との三つ巴で、社内データ接続の権限設計とガバナンスを提案の前提に置くべき。

- **政府レビュー解除後、24時間かけて全世界へ段階展開（OpenAI 公式、7/9）**: 6/26 の限定プレビュー（政府承認の約20組織）を経て、7/9 に ChatGPT・Codex・API で GA。Plus/Pro/Business/Enterprise は Sol を medium 以上の effort 設定で利用可、Pro/Enterprise は最高品質の GPT-5.6 Pro も選択可。Enterprise API は 7/7、**ChatGPT Plus は 7/10**、Team 全面は 7/14 の順で開放。命名は「数字＝世代、Sol/Terra/Luna＝能力ティア（各自のペースで進化）」。料金据え置き（Sol $5/$30、Terra $2.5/$15、Luna $1/$6）。
  - https://x.com/OpenAI/status/2075271435573244008

## モデル：SpaceXAI が Grok 4.5 を公開 — コーディング特化・Cursor 搭載・「Opusクラスを低価格で」

- **コスト高騰する AI コーディングを狙い撃ち（InfoWorld / Axios、7/8-9）**: Grok 4.5 はコーディング・エージェント用途に特化。80 tokens/秒、一部の SWE タスクで消費トークンが少ない。価格は **入力$2／出力$6（100万トークン、キャッシュ入力$0.50）**。ベンチ試算では1コーディングタスク当たり **Grok 4.5 $2.49 対 GPT-5.5(Codex) $5.07・Fable 5(Claude Code) $11.80**。SpaceXAI コンソール・Grok Build に加え **Cursor（Anysphere）に搭載** — API 競争でなく既存の開発環境経由で開発者に到達。EU は7月中旬提供予定。「Opusクラスを低コストで」と主張するが独立検証は未公開。
  - https://www.infoworld.com/article/4194895/spacexai-launches-grok-4-5-touts-lower-coding-task-costs-than-ai-rivals.html
  - https://x.ai/news/grok-4-5
- **コンサル観点**: フロンティア各社が同一週に投入し、なかでも Grok 4.5 は「Cursor 経由の流通」と「タスク単価の明示」で効率重視トレンドに乗る。**AI コーディング支援の提案では、モデル単体性能でなく "1タスク当たりコスト × 既存ツール（Cursor 等）への統合" で比較する枠組みが必要**。

## 開発者ツール：MCP に Enterprise-Managed Authorization（EMA）拡張が安定版

- **アクセスを一括管理（Publickey、7/7）**: Model Context Protocol にエンタープライズ向けの「Enterprise-Managed Authorization（EMA）extension」が安定版として提供開始。組織が MCP 経由のアクセスを中央で一括管理できる。ChatGPT Work / Claude Cowork のようなアプリ横断エージェントの普及に伴い、MCP 接続の権限統制が実運用課題として前面化。
  - https://www.publickey1.jp/
